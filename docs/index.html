On-Chain Dynamics Data Model

Welcome to the On-Chain Dynamics (OCD) Data Model documentationâ€”a comprehensive schema designed for managing user interactions, transactions, and statistics within the OCD platform. This model supports tracking transactions, user statistics, global platform metrics, referral programs, wallet management, and more. It ensures real-time synchronization of user activity and platform metrics through DynamoDB Streams and AWS Lambda functions.

Overview

The On-Chain Dynamics Data Model provides a robust foundation for decentralized finance (DeFi) applications. It is optimized for scalability, real-time processing, and seamless integration with blockchain operations. By leveraging Amazon DynamoDB, the model ensures high availability, efficient querying, and robust performance for blockchain-centric platforms.

Key Features

  • Real-Time Updates: Synchronizes user activity and platform metrics using DynamoDB Streams and Lambda functions.
  • Referral Tracking: Tracks referral codes, user sign-ups, trade volumes, and performance metrics.
  • User Customization: Supports personalized settings for sniper tools, wallet management, and token watchlists.
  • Access Control: Implements beta programs and waitlist features to dynamically manage user access levels.
  • Trade Flow Automation: Automatically updates user, referral, and platform-wide statistics upon every trade submission.
  • Scalability: Handles thousands of concurrent users and transactions seamlessly.

Tables

1. Transactions

  • ocdTransactions: Tracks user trades, including details about trades and blockchain interactions.
    • Trade types (buy/sell)
    • Blockchain and token pair details
    • Trade volume and value in ETH/USD

2. User Statistics

  • ocdUserStats: Aggregates and manages metrics for individual users.
    • Total trade volume and value
    • Trade counts and referral performance
    • User engagement metrics

3. Global Statistics

  • ocdTotalStats: Consolidates platform-wide metrics for analytics and reporting.
    • Total trade volume across the platform
    • User sign-ups and referral activity
    • Accumulated user points and leaderboard data

4. Referral Program

  • ocdReferrals: Manages referral-related data and performance tracking.
    • Referral codes and associated users
    • Trade volumes from referred users
    • Key Opinion Leader (KOL) prioritization for early access

5. Beta Users & Access Control

  • ocdBetaUsers: Controls platform access and manages onboarding processes.
    • Beta program enrollments
    • User access levels (e.g., full, restricted, trial)
    • Waitlist and onboarding statuses

6. User Settings

  • ocdUsers: Stores user-defined preferences for platform features.
    • Configurations for sniper tools (Ethereum, Solana, Tron)
    • Token watchlists and other preferences
    • Notifications and alert settings

7. Wallet Management

  • ocdWallets: Organizes user wallets with multi-chain support.
    • Blockchain type (e.g., Ethereum, Solana)
    • Encrypted private keys
    • User-defined visibility settings for wallets

8. Watchlist

  • ocdWatchlist: Tracks tokens and pools of interest to users.
    • Favorited tokens and their metadata
    • Liquidity pool data
    • Visual representation of tokens (e.g., icons)

9. Whitelist

  • ocdWhitelist: Tracks users who signed up via the main site popup.
    • User email or wallet addresses
    • Date and time of signup
    • Source of signup (e.g., marketing campaigns)

10. Waitlist

  • ocdWaitlist: Tracks users who signed up via the app signup page.
    • User email or wallet addresses
    • Date and time of signup
    • Status in the waitlist queue

Use Cases

  1. User Interaction Tracking: Monitor user activity and engagement across the platform in real-time.
  2. Referral Program Management: Efficiently handle referral-based user acquisition strategies and performance metrics.
  3. Custom Trading Configurations: Enable users to manage their personalized trading preferences, including sniper tools and watchlists.
  4. Platform Performance Analysis: Aggregate platform-wide data to identify trends and optimize system performance.
  5. Dynamic User Access: Manage access levels for beta users, waitlists, and feature rollouts dynamically.
  6. Wallet and Token Management: Securely store and manage user wallets and track user-favorited tokens.
  7. Seamless Scalability: Support a high volume of concurrent users and transactions while maintaining robust performance.

Technology Stack

  • Amazon DynamoDB: The core database for storing transactional, user, and global statistics data with high availability and low latency.
  • DynamoDB Streams: Enables real-time processing of table updates to synchronize data changes across the platform.
  • AWS Lambda: Processes DynamoDB Stream events to update statistics, referral metrics, and other derived data in real-time.
  • AWS IAM: Provides fine-grained access control and security for DynamoDB and Lambda interactions.
  • Encryption: Ensures sensitive data, such as private keys and user-specific configurations, is securely stored and transmitted.

Scalability and Optimization

The model is designed to handle thousands of concurrent users and transactions. By leveraging the strengths of DynamoDB and AWS Lambda, the On-Chain Dynamics platform provides:

  • High Availability: Ensures reliable access to data across multiple regions.
  • Efficient Querying: Optimized schema for low-latency operations.
  • Real-Time Processing: Processes data changes instantly with DynamoDB Streams and Lambda functions.
  • Flexibility: Supports dynamic user configurations and multi-chain compatibility.
